准备工作

What：net/http 是 Go 标准库里的 HTTP 客户端和服务端包。本教程聚焦服务端。

Why：Gin、Echo、Fiber 等框架底层都绕不开 HTTP 基础。先学标准库，后面用框架会更稳。

螺旋式学习规则

  1. 每一步只新增一个概念。
  2. 每一步都能独立复制为 main.go 运行。

Step 1 · 新概念：最小 HTTP 服务器

1. 最小服务器

What：一个 HTTP 服务至少需要一个 handler 和一个监听端口。

Why：handler 是请求进入业务代码的入口，ListenAndServe 负责监听端口并接收连接。

Principle：http.HandleFunc 把路径注册到默认路由表，http.ListenAndServe 使用默认路由表处理请求。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	// 注册根路径处理函数：浏览器或 curl 访问 / 时会执行这里。
	http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
		// ResponseWriter 用来写响应内容。
		fmt.Fprintln(w, "hello net/http")
	})

	// 启动 HTTP 服务，监听本机 8080 端口。
	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", nil); err != nil {
		panic(err)
	}
}

Step 2 · 新概念：多个路由

2. 多路由

What：路由负责把不同 URL 路径分发到不同 handler。

Why：真实服务通常有多个接口，例如健康检查、用户接口、订单接口。

Principle：ServeMux 是标准库路由器。这里开始显式创建 mux，不再依赖默认全局路由表。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	// 显式创建路由器，避免大型项目里使用全局默认路由表。
	mux := http.NewServeMux()

	// 健康检查路由。
	mux.HandleFunc("/health", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "ok")
	})

	// 用户路由。
	mux.HandleFunc("/users", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "users list")
	})

	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}

Step 3 · 新概念：请求方法

3. 请求方法

What：HTTP 方法描述动作，例如 GET 查询资源，POST 创建资源。

Why：同一个路径可能要根据方法做不同事情，比如 GET /users 查询，POST /users 创建。

Principle：标准库的 Request.Method 保存请求方法。Go 1.22 以后，ServeMux 也支持在 pattern 里写方法，例如 GET /users。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	mux := http.NewServeMux()

	// Go 1.22+ 的 ServeMux pattern 可以包含 HTTP 方法。
	mux.HandleFunc("GET /users", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "list users")
	})

	// 同一个路径，不同方法，映射到不同处理逻辑。
	mux.HandleFunc("POST /users", func(w http.ResponseWriter, r *http.Request) {
		w.WriteHeader(http.StatusCreated)
		fmt.Fprintln(w, "create user")
	})

	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# 查询用户
curl -i http://localhost:8080/users

# 创建用户
curl -i -X POST http://localhost:8080/users

Step 4 · 新概念：查询参数

4. 查询参数

What：查询参数是 URL 中 ? 后面的键值，例如 /search?q=go。

Why：查询参数适合表达过滤、排序、分页和搜索关键词。

Principle：r.URL.Query() 返回 url.Values，可以用 Get 读取单个参数。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	mux := http.NewServeMux()

	mux.HandleFunc("GET /search", func(w http.ResponseWriter, r *http.Request) {
		// 从 URL 查询参数中读取 q。
		keyword := r.URL.Query().Get("q")
		if keyword == "" {
			http.Error(w, "missing query parameter: q", http.StatusBadRequest)
			return
		}

		fmt.Fprintf(w, "search keyword: %s\n", keyword)
	})

	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# 带查询参数
curl -i "http://localhost:8080/search?q=golang"

# 不带查询参数，会返回 400
curl -i "http://localhost:8080/search"

Step 5 · 新概念：路径参数

5. 路径参数

What：路径参数把 URL 的一部分当成变量，例如 /users/42 里的 42。

Why：路径参数适合定位某个具体资源。

Principle：Go 1.22+ 的 ServeMux 支持 {name} 路径变量，并通过 r.PathValue("name") 读取。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	mux := http.NewServeMux()

	// {id} 是路径参数，匹配 /users/1、/users/abc 等路径。
	mux.HandleFunc("GET /users/{id}", func(w http.ResponseWriter, r *http.Request) {
		id := r.PathValue("id")
		fmt.Fprintf(w, "user id: %s\n", id)
	})

	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# 读取路径参数 id

Step 6 · 新概念：状态码与 Header

6. 状态码与 Header

What：状态码说明请求结果，Header 说明响应元信息。

Why：客户端不能只看响应文本，必须通过状态码和 Header 理解服务端意图。

Principle：先设置 Header，再调用 WriteHeader 或写响应体。第一次写响应体会隐式发送 200 OK。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

func main() {
	mux := http.NewServeMux()

	mux.HandleFunc("POST /users", func(w http.ResponseWriter, r *http.Request) {
		// Header 必须在 WriteHeader 或写 body 前设置。
		w.Header().Set("Content-Type", "text/plain; charset=utf-8")
		w.Header().Set("X-Request-Handled-By", "net-http-demo")

		// 创建资源通常返回 201 Created。
		w.WriteHeader(http.StatusCreated)
		fmt.Fprintln(w, "user created")
	})

	fmt.Println("server listening on http://localhost:8080")
	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# -i 会显示响应头和状态码
curl -i -X POST http://localhost:8080/users

Step 7 · 新概念：JSON 响应

7. JSON 响应

What：Web API 常用 JSON 作为响应格式。

Why：JSON 能被前端、移动端、其他服务稳定解析，比纯文本更适合结构化数据。

Principle：设置 Content-Type: application/json，然后用 json.NewEncoder(w).Encode(value) 写响应。

go · main.go
package main

import (
	"encoding/json"
	"net/http"
)

type User struct {
	ID   int64  `json:"id"`
	Name string `json:"name"`
}

func main() {
	mux := http.NewServeMux()

	mux.HandleFunc("GET /users/{id}", func(w http.ResponseWriter, r *http.Request) {
		// 读取路径参数，构造一个演示用户。
		id := r.PathValue("id")
		user := User{ID: 1, Name: "user-" + id}

		// JSON API 应明确设置响应类型。
		w.Header().Set("Content-Type", "application/json; charset=utf-8")
		_ = json.NewEncoder(w).Encode(user)
	})

	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# 请求 JSON 响应
curl -i http://localhost:8080/users/42

Step 8 · 新概念：JSON 请求体

8. JSON 请求体

What：请求体保存客户端提交的数据，例如创建用户时提交的 JSON。

Why：复杂数据不适合放进 URL，应放在请求体里。

Principle：使用 json.NewDecoder(r.Body).Decode 读取请求体，并用 MaxBytesReader 限制最大体积，避免大请求拖垮服务。

go · main.go
package main

import (
	"encoding/json"
	"net/http"
)

type CreateUserRequest struct {
	Name string `json:"name"`
}

type CreateUserResponse struct {
	ID   int64  `json:"id"`
	Name string `json:"name"`
}

func main() {
	mux := http.NewServeMux()

	mux.HandleFunc("POST /users", func(w http.ResponseWriter, r *http.Request) {
		// 限制请求体最大 1MB，防止客户端提交超大 body。
		r.Body = http.MaxBytesReader(w, r.Body, 1<<20)
		defer r.Body.Close()

		var req CreateUserRequest
		if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
			http.Error(w, "invalid json body", http.StatusBadRequest)
			return
		}
		if req.Name == "" {
			http.Error(w, "name is required", http.StatusBadRequest)
			return
		}

		w.Header().Set("Content-Type", "application/json; charset=utf-8")
		w.WriteHeader(http.StatusCreated)
		_ = json.NewEncoder(w).Encode(CreateUserResponse{ID: 1001, Name: req.Name})
	})

	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证

Step 9 · 新概念：Handler 抽象

9. Handler 抽象

What：http.Handler 是一个接口，核心方法是 ServeHTTP(ResponseWriter, *Request)。

Why：函数适合小例子，结构体 handler 适合真实项目，因为它可以保存依赖，例如数据库、服务对象、配置。

Principle：http.HandlerFunc 是一个适配器，把普通函数变成 http.Handler。

go · main.go
package main

import (
	"fmt"
	"net/http"
)

type UserHandler struct {
	// 真实项目里这里可以放 UserService、DB、Logger 等依赖。
	prefix string
}

func (h *UserHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
	// 结构体方法实现 ServeHTTP 后，就是一个 http.Handler。
	fmt.Fprintf(w, "%s users endpoint\n", h.prefix)
}

func main() {
	mux := http.NewServeMux()

	// mux.Handle 接收 http.Handler。
	mux.Handle("GET /users", &UserHandler{prefix: "demo"})

	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}
shell · 验证
# 请求结构体 handler
curl http://localhost:8080/users

Step 10 · 新概念：中间件

10. 中间件

What：中间件是包装 handler 的函数，通常签名是 func(http.Handler) http.Handler。

Why：日志、认证、限流、CORS、链路追踪都不应该散落在每个业务 handler 里。

Principle：中间件在调用 next.ServeHTTP(w, r) 前后插入通用逻辑。

go · main.go
package main

import (
	"fmt"
	"log"
	"net/http"
	"time"
)

func logging(next http.Handler) http.Handler {
	return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
		// next 前记录开始时间。
		start := time.Now()

		// 调用下一个 handler。
		next.ServeHTTP(w, r)

		// next 后记录耗时。
		log.Printf("%s %s cost=%s", r.Method, r.URL.Path, time.Since(start))
	})
}

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /hello", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "hello with middleware")
	})

	// 用 logging 包装整个 mux。
	if err := http.ListenAndServe(":8080", logging(mux)); err != nil {
		panic(err)
	}
}

Step 11 · 新概念：Request Context

11. Context

What：r.Context() 表示请求生命周期。

Why：客户端断开连接、请求超时、服务关闭时，下游工作应该能及时停止。

Principle：把 r.Context() 继续传给数据库、RPC、耗时函数；耗时函数用 select 监听 ctx.Done()。

go · main.go
package main

import (
	"context"
	"fmt"
	"net/http"
	"time"
)

func slowWork(ctx context.Context) error {
	// 模拟一个耗时操作。
	select {
	case <-time.After(2 * time.Second):
		return nil
	case <-ctx.Done():
		return ctx.Err()
	}
}

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /slow", func(w http.ResponseWriter, r *http.Request) {
		// 给单个业务操作增加 500ms 预算。
		ctx, cancel := context.WithTimeout(r.Context(), 500*time.Millisecond)
		defer cancel()

		if err := slowWork(ctx); err != nil {
			http.Error(w, "work canceled: "+err.Error(), http.StatusGatewayTimeout)
			return
		}

		fmt.Fprintln(w, "slow work done")
	})

	if err := http.ListenAndServe(":8080", mux); err != nil {
		panic(err)
	}
}

Step 12 · 新概念：http.Server 配置

12. Server 配置

What：http.Server 是可配置的服务端对象。

Why：生产环境不应该直接裸用 http.ListenAndServe，因为你需要配置读写超时、空闲连接超时、路由器等。

Principle：ListenAndServe 是便捷函数；自定义 http.Server 能设置更多安全和稳定性参数。

go · main.go
package main

import (
	"fmt"
	"net/http"
	"time"
)

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /hello", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "server configured")
	})

	server := &http.Server{
		Addr:              ":8080",
		Handler:           mux,
		ReadHeaderTimeout: 2 * time.Second,  // 限制读取请求头时间。
		ReadTimeout:       5 * time.Second,  // 限制读取整个请求时间。
		WriteTimeout:      10 * time.Second, // 限制写响应时间。
		IdleTimeout:       60 * time.Second, // 限制 keep-alive 空闲时间。
	}

	fmt.Println("server listening on http://localhost:8080")
	if err := server.ListenAndServe(); err != nil {
		panic(err)
	}
}

Step 13 · 新概念：优雅关闭

13. 优雅关闭

What：优雅关闭是在收到退出信号后，停止接收新请求，并等待正在处理的请求完成。

Why：直接杀进程可能中断正在执行的请求，造成部分写入、连接错误或用户体验问题。

Principle：server.Shutdown(ctx) 会关闭监听器、停止接收新连接，并等待现有请求完成，直到 context 超时。

go · main.go
package main

import (
	"context"
	"errors"
	"fmt"
	"net/http"
	"os"
	"os/signal"
	"syscall"
	"time"
)

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /hello", func(w http.ResponseWriter, r *http.Request) {
		fmt.Fprintln(w, "hello graceful shutdown")
	})

	server := &http.Server{
		Addr:              ":8080",
		Handler:           mux,
		ReadHeaderTimeout: 2 * time.Second,
	}

	// 在独立 goroutine 中启动服务，主 goroutine 用来监听退出信号。
	go func() {
		fmt.Println("server listening on http://localhost:8080")
		if err := server.ListenAndServe(); err != nil && !errors.Is(err, http.ErrServerClosed) {
			panic(err)
		}
	}()

	// 等待 Ctrl+C 或系统终止信号。
	stop := make(chan os.Signal, 1)
	signal.Notify(stop, os.Interrupt, syscall.SIGTERM)
	<-stop

	// 给正在处理的请求最多 5 秒完成时间。
	ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
	defer cancel()

	if err := server.Shutdown(ctx); err != nil {
		panic(err)
	}
	fmt.Println("server stopped gracefully")
}

Step 14 · 新概念：httptest

14. 测试 Handler

What：net/http/httptest 用于在测试里构造请求和记录响应。

Why：HTTP handler 是业务入口，必须能自动化测试，而不是每次手动 curl。

Principle：httptest.NewRequest 构造请求，httptest.NewRecorder 捕获响应。

go · main.go
package main

import (
	"encoding/json"
	"net/http"
)

type HealthResponse struct {
	Status string `json:"status"`
}

func healthHandler(w http.ResponseWriter, r *http.Request) {
	// 测试目标：返回 JSON 健康检查结果。
	w.Header().Set("Content-Type", "application/json; charset=utf-8")
	_ = json.NewEncoder(w).Encode(HealthResponse{Status: "ok"})
}

func main() {
	mux := http.NewServeMux()
	mux.HandleFunc("GET /health", healthHandler)
	_ = http.ListenAndServe(":8080", mux)
}
go · main_test.go
package main

import (
	"net/http"
	"net/http/httptest"
	"strings"
	"testing"
)

func TestHealthHandler(t *testing.T) {
	// 构造一个 GET /health 请求。
	req := httptest.NewRequest(http.MethodGet, "/health", nil)

	// Recorder 用来捕获 handler 写出的响应。
	rec := httptest.NewRecorder()

	// 直接调用 handler，不需要真正启动端口。
	healthHandler(rec, req)

	if rec.Code != http.StatusOK {
		t.Fatalf("expected status 200, got %d", rec.Code)
	}

	if !strings.Contains(rec.Body.String(), `"status":"ok"`) {
		t.Fatalf("unexpected body: %s", rec.Body.String())
	}
}

常见错误

裸用默认 mux

小例子可以用 http.HandleFunc，真实项目更推荐显式创建 http.NewServeMux()，避免全局状态混乱。

先写 body 后设 Header

第一次写响应体会发送默认状态码和 Header。Header 应在写 body 或 WriteHeader 前设置。

不限制请求体大小

读取 JSON body 前使用 http.MaxBytesReader，否则恶意大请求可能占用大量内存或连接时间。

没有 Server 超时

生产环境应配置 ReadHeaderTimeout、ReadTimeout、WriteTimeout 和 IdleTimeout。

忽略 Request.Context

数据库、RPC、耗时函数都应接收并传递 r.Context()，让取消和超时能向下游传播。

缺少自动化测试

handler 可以用 httptest 快速测试，不需要真正启动端口，反馈更快也更稳定。

Practice：练习题

1. Todo 列表

实现 GET /todos 返回 JSON 数组，只用内存 slice 保存数据。

3. 查询 Todo

Interview Questions：面试题

1. http.Handler 和 http.HandlerFunc 有什么区别？

http.Handler 是接口，要求实现 ServeHTTP。http.HandlerFunc 是函数类型，并实现了 ServeHTTP，所以普通函数可以被适配成 handler。

2. 为什么生产环境不建议只用 http.ListenAndServe？

便捷函数无法配置完整的超时策略。生产环境应使用 http.Server，配置 ReadHeaderTimeout、ReadTimeout、WriteTimeout、IdleTimeout，并实现优雅关闭。

3. WriteHeader 和 Write 的顺序有什么讲究？

Header 应先设置。WriteHeader 会发送状态码；如果先调用 Write，Go 会隐式发送 200 OK，之后再改状态码就无效了。

4. Request.Context 在 HTTP 服务里有什么用？

它表示请求生命周期。客户端断开、服务关闭或上游取消时，context 会被取消。下游数据库、RPC 和耗时函数应该接收它，以便及时停止无意义工作。

5. 中间件的本质是什么？

中间件是对 http.Handler 的包装函数，典型签名是 func(http.Handler) http.Handler。它通过调用 next.ServeHTTP 把请求传给下一个 handler。

6. 如何测试 HTTP handler？

使用 net/http/httptest。httptest.NewRequest 构造请求，httptest.NewRecorder 捕获响应，然后直接调用 handler 并断言状态码、Header 和 body。

总结

这套螺旋式路径可以压缩为一条成长线：先会启动服务，再会分发请求，然后会读输入、写输出，接着用 handler 和 middleware 管理结构，最后补上 context、server timeout、优雅关闭和测试。

最终口诀

Handler 是入口，Request 是输入，ResponseWriter 是输出，ServeMux 是分发，Server 是生产边界，Context 是生命周期。
